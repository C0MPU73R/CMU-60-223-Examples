PureData Starter: Overview

This section is an introduction to physical computing with the real-time data-flow programing language PureData. Such environments take their name ("data-flow") after their visual analogy of graphical modules that intake and process data and pass it along to other modules.

Takeaway Lessons

  • PureData offers tremendous flexibility: one one environment you can work with physical computing, audio analysis and syntehsis, video analysis and synthesis and more. PureData is free and open source, and it works on personal computers (Windows, OS X, Linux), smart phones (iOS, Android), embedded platforms (Raspberry Pi, Udoo, BeagleBone)
  • PureData is good at: rapid prototyping of interactive audio/visual systems, controlling multiple parallel processes with complex timing; audio analysis and synthesis,
  • PureData is bad at: recurssive programming, graphical user interfaces, intergration of documentation, following computer science conventions for naming
  • PureData is very extensible: you can extend PureData's functionality with C/C++, Python or a range of other text-based programming languages

Lab Exercises

The lab exercises are intended to be performed by pairs of students. These will familiarize you with processes you have already explored with micro-controllers, but now with a personal computer or Raspberry Pi. They will also familiarize you with the work flow for using PureData on a laptop to communicate with a micro-controller, as well as PureData on an Raspberry Pi

Graded Project: Dream Machine

The graded project is inspired by the Dreammachine, "a stroboscopic flicker device that produces visual stimuli. Artist Brion Gysin and William S. Burroughs's "systems adviser" Ian Sommerville created the dreamachine after reading William Grey Walter's book, The Living Brain." The goal is to make an enchanted object that uses sound, light and movement to create a dream-like state for the viewer.

Challenge Yourself

Any one who completes the basic exercises should consider undertaking some of the optional challenge exercises.

  1. Create a automaton percussive instrument that performs changing rhythmic patterns
  2. Create a PureData based client-server system where a central server running PureData sends messages to numerous client computers running PureData, which in turn generate sound or actuate kinetic elements
